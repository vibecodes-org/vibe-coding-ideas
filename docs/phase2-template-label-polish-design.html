Phase 2: Template Label Polish

Design proposals — building on the UX Audit findings (P1-P4)

Proposal 1: Auto-Rules Empty State Tip Card

Addresses P2 Auto-rules empty state too subtle and P4 No "get started" banner.

Current (before)
⚡ Workflow Triggers
Automatically apply a workflow when a task gets a specific label.

No workflow triggers yet. When a label is applied to a task, a workflow trigger can automatically attach a workflow template.

Proposed (after)
⚡ Workflow Triggers
Automatically apply a workflow when a task gets a specific label.
💡
Connect this template to a label

Add a workflow trigger so this template is automatically applied when you label a task. No manual step needed.

Design rationale:
- Uses the same dashed-border + amber color scheme as the sidebar empty state for visual consistency
- Lightbulb icon signals "tip" without being intrusive
- Inline "Add trigger" button removes the need to find the small "+ Add" in the header
- Clicking "Add trigger" opens the same inline add-rule form that "+ Add" opens
- Once a rule exists, the tip card disappears entirely (replaced by the rule list)

Proposal 2: Post-Creation Toast with Hint

Addresses P1 No post-creation nudge.

Proposed toast

Design rationale:
- Zero additional UI complexity — just a subtitle line on the existing toast
- Points users toward the auto-rules section they're likely to miss below the fold
- Disappears after the standard toast timeout — not persistent or blocking
- Same hint text shown after import (via ImportTemplateLibraryDialog) when auto-wire was unchecked

Keyword mapping

Uses the same label suggestions as the library template backfill from Phase 1. Match is case-insensitive against the template name.

Design rationale:
- Purely informational — no action button, no picker. Users read it and move on.
- Appears only when a keyword matches. No hint = no distraction for custom workflows.
- Uses actual label badge styling (colored pill) so users can mentally connect to the board's label system.
- The hint says "after creating" — it sets the right expectation without adding steps to the dialog.
- Keyword map is defined as a simple constant — no AI, no API call, instant match.

Implementation Summary

Proposal Pain Point File(s) Effort

Accessibility

  • Tip card uses role="note" for screen readers
  • "Add trigger" button is keyboard-focusable with visible focus ring
  • Inline hint uses sufficient amber-on-dark contrast (WCAG AA for text at this size)
  • Label badges in hints have white text on saturated backgrounds (same as board labels)
  • Toast hint is announced via sonner's existing ARIA live region

Not included (deferred)

  • Full label picker in CreateTemplateDialog — too complex for this phase
  • Persistent banner on template detail — tip card in auto-rules section is sufficient
  • Dismiss/remember for tip card — it disappears once a rule is added, which is the natural completion
