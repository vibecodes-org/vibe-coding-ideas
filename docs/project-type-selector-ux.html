Design Decision

The preview is not collapsible on the Create Idea form — users should see what they're getting before committing. On the Workflows tab dialog (Mockup 3), sections are collapsible since users are more experienced at that point.

✘ Failed to apply kit. Your idea was created — you can apply a kit from the Workflows tab.
Error Recovery

If kit application fails, the idea is still created (it's a two-step process: create idea → apply kit). The user is redirected to the idea detail page (not board) with an error toast. They can retry from the Workflows tab. This is graceful degradation — never lose the user's title and description.

Responsive Mobile Breakpoints

The selector grid and preview adapt to screen width.

Breakpoints: Grid uses auto-fill, minmax(200px, 1fr) — naturally stacks to 1-2 columns on mobile. Dialog uses minmax(160px, 1fr) for tighter space.

Mobile Dialog Behaviour

  • Below 768px, the Apply Kit dialog becomes a bottom sheet (slides up from bottom, full width)
  • Kit cards use smaller padding and icon size
  • Preview shows a compact summary (agent count, step count, label count) instead of full role/step listing
  • Workflow step chips wrap naturally via flex-wrap

Data Flow

  • Create Idea page: Server component fetches kits via getActiveKits(), passes as prop to IdeaForm
  • Workflow steps for preview: Fetched client-side when a kit is selected (only if kit has workflow_library_template_id). Use a lightweight query or include step data in the kits query.
  • Submit with kit: createIdea() returns new idea ID → applyKit(ideaId, kitId) → redirect
  • Workflows tab dialog: getActiveKits() fetched on dialog open, applyKit() on confirm

Keyboard & Accessibility

  • Kit cards are focusable (tabIndex={0}) with role="radio" in a role="radiogroup"
  • Arrow keys navigate between cards, Enter/Space selects
  • Kit preview announced via aria-live="polite" region
  • Dialog uses role="dialog" with focus trap and Escape to close
  • All interactive elements meet WCAG 2.1 AA contrast (4.5:1 for text, 3:1 for UI controls)
