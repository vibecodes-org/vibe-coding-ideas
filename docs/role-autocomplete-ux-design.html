3. Suggestion Grouping & Sources

Suggestions are grouped by source to help users understand where each role comes from. Groups appear in priority order:

GroupSourceWhen shownBadge
Idea PoolAgents allocated to the current ideaWhen ideaId prop is providedpool + agent name
My AgentsUser's own bot profilesAlways (agent dialogs)mine
Standard Roles8 BOT_ROLE_TEMPLATESAlways (as fallback)None (implicit)
Deduplication: If a role appears in multiple groups, show it in the highest-priority group only. E.g., if "Developer" is both a pool agent role and a standard role, show it only under "Idea Pool".
Compact variant simplification: In the compact template step context, collapse to just two groups: "Idea Agents" (pool) and "Standard". Skip "My Agents" since template step roles should match idea pool agents, not the template author's personal agents.

7. Data Fetching Strategy

Principle: Minimize network requests. Standard roles are static constants. User agent roles and idea pool roles are fetched once per dialog mount, not per keystroke.
Data sourceFetch methodCache
Standard rolesImport from BOT_ROLE_TEMPLATESStatic, no fetch needed
User's agent rolesSupabase query on mount: bot_profiles where owner_id = user.idPer dialog session
Idea pool rolesSupabase query on mount: idea_agents join bot_profiles where idea_idPer dialog session

9. Design Decisions & Rationale

  • Popover + Command (not a custom dropdown): Leverages existing shadcn/ui primitives already in the project. cmdk handles filtering, keyboard navigation, and accessibility out of the box.
  • Groups with badges: Users need to understand why a role is suggested. "Pool" roles mean "this will auto-match". "Mine" roles mean "I already use this". "Standard" roles are just common suggestions.
  • Agent name in pool badges: In the step detail dialog, showing which agent will match (e.g., "Atlas") gives immediate feedback on the fuzzy matching system.
  • No enforcement: The combobox never blocks custom input. This preserves the flexibility of the fuzzy matching system identified in the Requirements step.
  • Helper text on agent dialogs, tooltip on step roles: Different surfaces have different space budgets. Agent dialog role fields have room for inline helper text. Compact step role inputs use a (?) tooltip to avoid cluttering the row layout.
  • Two groups in compact mode: Template steps match against idea pool agents, so "My Agents" is noise in that context. Simplifying to "Idea Agents" + "Standard" reduces cognitive load.
