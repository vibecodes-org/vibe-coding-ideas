The Core Problem

VibeCodes has three powerful systems (Agents, Workflows, Boards) that work brilliantly together — but users encounter them as three separate islands with no clear bridge between them. A new user creates an idea, lands on an empty board, and has no guidance connecting them to agents or workflows. The app never tells them: "Here's the complete picture of how this all works together."

Key Themes

  • The "Empty Board Cliff" — After onboarding, users land on an empty board with no tasks. The AI Generate button exists but isn't automatically prompted. Users who dismiss the empty state see bare columns with no idea what to do.
  • Agent-Workflow Disconnect — Agents and workflows are deeply connected (agents execute workflow steps) but the UI treats them as separate features. Users can create workflows without agents, create agents without ideas, and never learn how they connect.
  • Progressive Disclosure Gaps — The app has good empty states individually, but they don't chain together. After completing one action, there's rarely a "next step" nudge pointing to the logical follow-up.
  • Concept Overload — New users encounter Ideas, Boards, Columns, Tasks, Labels, Workflows, Templates, Auto-Rules, Agents, Roles, Teams, and MCP within their first session. No explanation builds up this mental model incrementally.
  • Help Content is Orphaned — 9 guide pages exist at /guide but are never linked from the features they describe. No contextual "?" or "Learn more" links.

3 Onboarding Flow

The 5-step wizard in src/components/onboarding/onboarding-dialog.tsx is the strongest part of the new-user experience, but has key gaps.

What Works Well

  • Non-dismissable but skippable — ensures users see the onboarding without forcing completion of every step
  • Free AI enhance — lowers the barrier to trying AI features during idea creation, clearly labelled "Free"
  • Agent team quick-start — featured teams give users functional agents immediately without understanding the agent system
  • Visual polish — gradient accents, animated step indicator, confetti on completion
  • Value proposition cards on Step 0 quickly communicate what the app does

Findings

HIGH F1: Post-onboarding "View your board" leads to empty board Onboarding → Board

The success screen's primary CTA is "View your board" but the board is empty. Users who created an idea but didn't generate tasks land on a blank board with no explanation of what happened to their idea. The jump from "idea created with a description" to "empty kanban board" is jarring.

Auto-trigger AI task generation when the user clicks "View your board" after creating an idea in onboarding, or add an interstitial asking "Would you like AI to generate tasks from your idea?"

MEDIUM F2: Agent team step doesn't explain why agents matter Onboarding Step 2

Step 2 says "Start with a pre-built team of AI agents. You can customise them later." but doesn't explain what agents actually DO. Users unfamiliar with AI agent concepts may skip this step because the value isn't clear, then miss the entire agent system.

Add a brief explanation: "AI agents can automatically work on your tasks — reviewing code, writing tests, designing UX. Pick a team to get started." Show a micro-animation of an agent picking up and completing a task.

MEDIUM F3: No idea visibility choice during onboarding Onboarding Step 3

Ideas created during onboarding are hardcoded as visibility: "public". Users working on sensitive projects have no option to create a private idea during onboarding, potentially discouraging them from describing their idea in detail.

Add a visibility toggle (public/private) to the idea creation step with a brief explanation of what each means.

LOW F4: Tag suggestions are limited and hardcoded Onboarding Step 3

Only 6 tags are suggested (ai, web, mobile, tool, game, automation) with no option to type custom tags. Users building something that doesn't fit these categories (e.g., data science, IoT, education) have no way to categorize their idea during onboarding.

Add a text input for custom tags alongside the suggested pills, or expand suggestions based on popular community tags.

LOW F5: Dead code — WelcomeExperience can never render Dashboard

In dashboard/page.tsx, the WelcomeExperience component's condition (!onboardingCompleted && ideasCount === 0 && collaborationsCount === 0) is identical to isNewUser, which is checked first in the ternary. The WelcomeExperience branch is unreachable.

Remove the dead code branch or differentiate the conditions (e.g., WelcomeExperience for returning users who somehow bypassed onboarding).

4 Dashboard & First-Run Experience

The dashboard at src/app/(main)/dashboard/page.tsx is the user's home base but provides minimal guidance for new users.

CRITICAL F6: Post-onboarding dashboard is a wall of empty states Dashboard

After completing onboarding (even with an idea created), the dashboard shows: 4 stat cards (mostly zeros), empty Active Boards, empty My Tasks, a single idea in My Ideas, empty Collaborations, empty Activity. The onboarding checklist FAB is a small floating button in the corner — easy to miss. There is no prominent "What to do next" guidance.

The user's first impression after the polished onboarding wizard is a mostly-empty page with no clear next action. This is the highest-risk moment for user drop-off.

Replace the empty dashboard for new users (first 24 hours or until they have board tasks) with a guided first-run dashboard that prominently shows: (1) their idea with a "Generate Board" CTA, (2) their agent team with a "View Agents" link, (3) a 3-step visual showing "Create Idea → Generate Board → Agents Work on Tasks". Only switch to the standard dashboard once the user has board tasks.

HIGH F7: Conditionally hidden sections prevent feature discovery Dashboard

My Agents, Active Discussions, and Active Boards sections are completely hidden when empty. New users never learn these features exist from the dashboard. The sections only appear when the user has already engaged with the feature through another path.

Show all sections with contextual empty states that include a brief explanation + CTA. E.g., "Active Discussions — Plan features with your team before building" with a "Start a Discussion" button, rather than hiding the entire section.

MEDIUM F8: Onboarding checklist "Connect Claude Code" can never complete Dashboard Checklist

The checklist step for "Connect Claude Code" is hardcoded as done: false with no detection mechanism. Users who complete all other steps still see "1 remaining" on the checklist badge. The checklist FAB remains visible indefinitely (unless manually dismissed), creating a permanent sense of incompleteness.

Either detect MCP connection (e.g., check if the user's MCP client has made any API calls) or mark the step as complete once dismissed/viewed, or remove it from the required steps and show it as an optional "Pro tip" instead.

5 Idea Detail & Discovery

The idea detail page at src/app/(main)/ideas/[id]/page.tsx is the hub connecting all features, but the connections aren't visible enough.

CRITICAL F9: No "Getting Started" guidance on a new idea page Idea Detail

A newly created idea with no description, no agents, no discussions, and no board tasks shows a sparse page. There's no suggested workflow like "1. Flesh out your description, 2. Add agents, 3. Generate a board." The user has to already know what to do.

The "Board" button — the most important navigation for active projects — is a small outline button with minimal visual weight, easily overlooked between the hero card and discussions section.

For ideas with no board tasks, show a prominent "Getting Started" card below the hero with a visual 3-step flow: "Describe your idea → Add AI agents → Generate your board." Make the Board button larger and more prominent, or auto-redirect to the board for new ideas.

HIGH F10: Agent section is invisible when empty for team members Idea Detail — Agents

The Agents metadata strip shows the "+" button when the user has unallocated bots, but if the user has zero bots, there's no indication that agents exist or should be created. The section just shows "Agents" with nothing next to it. There's no "Create your first agent" nudge, no link to /agents, and no explanation of why agents matter for this idea.

When the agent pool is empty, show a subtle nudge: "No agents yet — AI agents can work on your board tasks. Create agents or browse the community." Keep it compact but actionable.

MEDIUM F11: Discussions not mentioned in onboarding or getting-started Idea Detail — Discussions

The discussions feature is never introduced during onboarding. On the idea page, the empty state says "Start a discussion to plan features..." but for solo users, this doesn't seem relevant. The feature's value for collaborative planning is undersold.

Include discussions in the idea "Getting Started" card for ideas with collaborators: "Have collaborators? Start a discussion to align on priorities."

6 Board, Tasks & AI Generation

The kanban board is the operational core of VibeCodes. The AI generation flow is polished, but the post-generation experience leaves users without guidance.

CRITICAL F12: No post-generation guidance — tasks created but no workflows applied Board — AI Generate

After AI generates tasks and populates the board, the dialog closes and the user sees their new tasks. But there's no prompt to: set up workflows, allocate agents, or understand what to do next. Tasks sit as static items with no automation unless the user independently discovers the Workflows tab, imports a template, creates auto-rules, AND allocates agents.

This is the single biggest coherence gap in the app. The AI generation is the "wow moment" but it immediately drops the user off a cliff.

After AI generates tasks, show a post-generation dialog: "Your board is ready! Next steps:" with (1) "Set up a workflow to automate tasks" → Workflows tab, (2) "Assign agents to work on tasks" → Agents tab. Or better: offer to auto-apply a workflow template and auto-allocate agents as part of the generation flow.

HIGH F13: Board empty state dismissal is permanent Board — Empty State

The "Show columns & add tasks manually" link at the bottom of the board empty state permanently dismisses it. If a user clicks this without fully reading the empty state, they can never get back to the AI Generate funnel or the agent/workflow nudge cards from the empty state view. The AI Generate button remains in the toolbar, but the contextual nudges are gone.

Make the dismissal temporary (session-based) or add persistent nudge cards to the board toolbar/sidebar when the board has zero workflows and zero allocated agents.

MEDIUM F14: "Apply Workflow" button in task detail is low visibility Board — Task Detail

The workflow section in the task detail dialog uses a dashed-border ghost button at the bottom of the Details tab. Users who don't scroll past the description will never see it. When there are no templates, the message "No templates — create one in the Workflows tab" doesn't link to the tab.

Make "Apply Workflow" more prominent — either move it above the description or use a more visible button style. The "no templates" message should be a direct link to the Workflows tab.

MEDIUM F15: No import mention in board empty state Board — Empty State

The CSV/JSON/bulk text import feature exists in the board toolbar but isn't mentioned in the empty state. Users migrating from other tools (Trello, Jira) would benefit from knowing this option exists upfront.

Add a secondary link in the empty state: "Or import tasks from another tool" linking to the import dialog.

LOW F16: 50-task AI generation cap is silent Board — AI Generate

AI generation silently truncates to 50 tasks via .slice(0, 50). Users requesting large boards won't know tasks were cut off.

Show a notice: "Generated 50 of ~N tasks (maximum per generation). Run again to generate more."

LOW F17: No inline preview of tasks before AI generation applies Board — AI Generate

Users can select/deselect generated tasks in the preview but can't edit titles, descriptions, or column assignments before applying. Minor friction for users who want to tweak the AI output.

Add inline title editing in the preview table (click-to-edit on the title cell).

7 Workflows & Auto-Rules

The Workflows tab in src/components/board/workflows-tab.tsx has excellent contextual help when empty but becomes harder to navigate as complexity grows.

What Works Well

  • "How workflows work" guide card — the 4-step explanation is the best piece of in-app education in the entire application
  • "Agents needed" warning — prevents users from creating templates they can't use
  • Template library import — one-click access to pre-built templates
  • Inline label creation in auto-rule form — users don't need to leave the workflow to create labels
HIGH F18: Workflow concept is never introduced outside the Workflows tab Workflows — Discoverability

Users encounter "Apply Workflow" in task details, "Set Up Workflows" in the board empty state, and "Workflows" in the tab bar — but there's no explanation of what workflows are until they visit the Workflows tab and see the "How workflows work" card. Users who apply a workflow from a task detail dialog have no prior context.

Add a tooltip or brief inline explainer wherever workflows are mentioned outside the Workflows tab. E.g., in the task detail dialog: "Workflows automate your process — agents pick up steps sequentially. Learn more"

HIGH F19: Auto-rules concept (label = trigger) is unintuitive Workflows — Auto-Rules

Auto-rules map labels to templates: "When label X is added, apply template Y." But most users think of labels as categories, not triggers. The mental model of "labelling a task causes automated workflow application" is non-obvious and only explained in-context within the Workflows tab when no rules exist.

Rename or reframe: instead of "Auto-Rules," consider "Workflow Triggers" with clearer language: "Automatically start this workflow when a task is labelled [X]." Add a one-line explainer in the auto-rules section header.

MEDIUM F20: No template preview when applying from task detail Workflows — Task Detail

The template select dropdown in TaskWorkflowSection shows template name and step count but no way to preview the actual steps before applying. Users must blindly choose and apply, then inspect what they got.

Add an expandable preview showing step titles, roles, and approval gates when hovering/clicking a template in the selector.

MEDIUM F21: Workflow step "Retry" is hover-only Workflows — Step UI

The inline "Retry" button on failed workflow steps only appears on hover. This is inaccessible on touch devices and violates WCAG interactive element guidelines.

Always show action buttons for failed/actionable steps, or use a secondary click target (e.g., right-click menu or kebab menu).

8 Agents & Teams

The Agents Hub at src/components/agents/agents-hub.tsx and idea-level agent allocation are well-built individually, but the connection between agents, ideas, and workflows needs reinforcement.

CRITICAL F22: No clear path from "I have agents" to "agents are doing work" Agents — End-to-End

The journey from agent creation to productive use requires 5 independent steps that the app never connects: (1) Create agent → (2) Allocate to idea → (3) Create workflow template → (4) Apply template to task → (5) Agent executes steps via MCP. Each step is individually documented but there's no guided path through all five. A user who adds a featured team during onboarding may never allocate those agents to their idea.

After adding a featured team or creating agents, prompt: "Now allocate agents to your ideas so they can work on tasks." On the idea page, when agents are allocated but no workflows exist: "Your agents are ready! Set up a workflow so they can start working." Create a visual pipeline showing the agent lifecycle.

CRITICAL F23: Agent match warning links to wrong location Agents — Workflow Steps

When workflow steps have roles with no matching agent, the TaskWorkflowSection shows a warning banner that links to /agents (the global Agents Hub) instead of ?tab=agents (the board's own Agents tab where they can immediately allocate agents to this idea). This sends users away from their current context.

Link to ?tab=agents instead of /agents. The board's Agents tab has the Role Coverage panel and "Add Agent" controls — it's the right place to fix the mismatch.

HIGH F24: Featured team "Add Team" doesn't explain what happens Agents — Featured Teams

Clicking "Add Team" clones the team's agents into the user's account. But the button label and surrounding UI don't explain that this creates copies of the agents in your account (not shared references). Users may be confused about ownership, especially if they later see the same agent names on the Community tab.

Change button text to "Clone Team to My Agents" or add a tooltip: "Creates copies of these agents in your account. You can customise them afterwards."

MEDIUM F25: Agent creation allows functionally useless agents Agents — Create Dialog

Only the agent name is required. A user could create an agent with just a name — no role, no system prompt, no skills. This agent would be useless for workflow matching (which relies on roles) and MCP execution (which relies on system prompts). No validation warns about this.

Add soft validation: when role or system prompt is empty, show an inline hint like "Add a role so this agent can be auto-assigned to workflow steps" and "Add a system prompt to define how this agent works."

9 Help, Nudges & Guidance System

VibeCodes has 9 guide pages and numerous empty-state nudges, but they operate in silos without connecting to each other.

HIGH F26: Guide pages are never linked from features Help System

Nine comprehensive guide pages exist at /guide/* (Getting Started, Kanban Boards, Workflows, AI Agent Teams, etc.) but are only accessible via the navbar "Guide" link. No feature in the app links to its relevant guide section. The Workflows tab doesn't link to /guide/workflows. The board doesn't link to /guide/kanban-boards. The agents page doesn't link to /guide/ai-agent-teams.

Add contextual "Learn more" or "?" icon links from each feature to its guide page. E.g., Workflows tab header: "Workflows (?)", linking to /guide/workflows. Board toolbar: "Need help? Board guide". Agents hub: "How agents work →"

HIGH F27: No in-app walkthrough for complex features Help System

The board, workflows, and agents pages have no interactive walkthrough or tour for first-time visitors. Users must figure out drag-and-drop, column management, filtering, workflow application, agent allocation, and MCP connection through exploration alone. The static guide pages help, but they require leaving the current context.

Implement a lightweight guided tour (e.g., using tooltips that highlight UI elements in sequence) for first-time visitors to the board page. Focus on: column structure, creating tasks, the Workflows tab, the Agents tab, and AI generation. Store completion in localStorage.

LOW F28: Guide pages have no inter-page navigation Help System

Each guide page is standalone with no sidebar or breadcrumbs to navigate between sections. Users must return to the guide hub to find another section.

Add a sidebar navigation or "Next/Previous" links at the bottom of each guide page.

12 Strategic Recommendations

Three high-level initiatives that would address the majority of findings and transform the new-user experience.

🚀
Initiative 1: The "Golden Path" — Guided First-Run Experience
Addresses: F1, F6, F9, F12, F22 (all 5 critical findings)

Create a seamless guided journey from onboarding through to "agents working on tasks." Instead of dropping users on an empty dashboard after onboarding, create a flow:

  1. Onboarding creates idea (existing) → immediately offers to generate tasks with AI
  2. After task generation, offer to import a workflow template and auto-allocate agents
  3. Show a "Your project is set up!" summary with the board, active workflows, and allocated agents
  4. Replace the empty dashboard with a first-run dashboard that prominently surfaces the user's idea, board, and agents with clear "Next action" prompts

This collapses the current 8-step journey into 2 steps after signup, getting users to the "wow moment" (agents working on tasks) within their first session.

🔗
Initiative 2: Cross-Feature Connective Tissue
Addresses: F7, F10, F13, F18, F23, F26, F27

Add persistent, contextual nudges that connect features to each other:

  • Dashboard: Show all sections (even when empty) with contextual CTAs explaining each feature
  • Idea page: "Getting Started" card when no board tasks exist; agent nudge when pool is empty
  • Board: Post-generation workflow/agent prompt; persistent sidebar nudges (not just empty-state)
  • Task detail: Workflow explainer inline; agent match warning linking to the board's Agents tab
  • Every feature header: "?" icon linking to the relevant guide page
  • Board first visit: Lightweight tooltip tour highlighting key areas

The goal is that every screen tells the user what to do next. No dead ends.

💡
Initiative 3: Concept Simplification & Terminology
Addresses: F2, F19, F24, F25

Reduce cognitive load by simplifying or reframing complex concepts:

  • Rename "Auto-Rules" to "Workflow Triggers" with clearer language about label-based activation
  • Explain agents as "AI team members" consistently — the onboarding does this but the rest of the app uses technical language
  • Featured team "Add Team" → "Clone Team to My Agents" with tooltip explaining the copy
  • Agent creation soft validation — nudge users to add role and system prompt with inline hints
  • Create a "VibeCodes in 60 seconds" visual explainer (Ideas → Board → Agents → Workflows → MCP) accessible from the landing page, guide, and dashboard
