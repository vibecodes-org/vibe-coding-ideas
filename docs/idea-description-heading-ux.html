Fix: Idea Description Visual Hierarchy

The description is the most important content on the idea page, but it currently has the weakest visual presence. Here's the problem and the fix.

Problem: Current Hierarchy BROKEN

Look at how sections compare right now:

The description label is tiny, faded, uppercase, and has no icon. Every other section (Discussions, Attachments, Comments) uses bold text with icons. The description — the primary content — looks like metadata.

Fix: Matching Hierarchy PROPOSED

Two changes: (1) give the description section the same heading style as other sections, and (2) move it ABOVE Discussions since it's the primary content.

Implementation

Change 1 — Description heading style (in inline-idea-body.tsx):

Replace the tiny text-[11px] uppercase text-muted-foreground/60 label with:

<h3 className="flex items-center gap-2 text-sm font-semibold">
  <FileText className="h-4 w-4" />
  Description
</h3>

This matches the exact pattern used by Attachments (line 345 of idea-attachments-section.tsx).

Change 2 — Move description above Discussions (in ideas/[id]/page.tsx):

The description card (lines 495-503) should move above the Discussions Preview (lines 485-493). The page order should be:

  1. Getting Started card (when no tasks)
  2. Description (primary content — MOVED UP)
  3. Discussions Preview
  4. Attachments
  5. Comments

Change 3 — Separate GitHub URL from Description (in inline-idea-body.tsx):

The GitHub URL and Description are currently inside the same component/card. They should be visually separated — GitHub URL gets its own small section or moves to the metadata area at the top. This prevents the GitHub URL from making the description card look cluttered.

Why this matters: The description is the first thing a new visitor reads to understand the idea. It should have the strongest visual presence after the title, not the weakest. The current tiny uppercase label makes it look like a form field label, not a content section.
