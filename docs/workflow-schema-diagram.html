Concurrency Guards (migration 00081)
Database Level
  • Partial unique index on workflow_runs(task_id) WHERE status NOT IN ('completed','failed') -- prevents multiple active runs per task
Application Level
  • .eq("status", "pending") on claim -- prevents double-claim
  • .eq("status", "in_progress") on complete -- prevents double-complete
  • .in("status", [...]) on fail -- prevents invalid transitions
  • .maybeSingle() returns null instead of error if guard blocks
  • checkAndCompleteRun() shared helper for consistent run completion
