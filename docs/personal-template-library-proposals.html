Personal Template Library

Design proposals for cross-idea workflow template reuse. Building on the Audit Current UX findings.

Proposal A: "Save to My Templates" Action

Pattern source: Asana's "Save as Template" from the project header dropdown and Monday.com's board menu "Save as Template". Both place the save action near the source item, not buried in settings.
Design decision: Place a persistent "Save to My Templates" button in the template detail header, next to Edit and Delete. This follows the proximity principle (action near the thing it acts on) and is always visible — no context menu hunting needed.
Terminology: We use "My Templates" instead of "Library" throughout. "Library" is an abstraction that describes the container; "My Templates" immediately tells users what it is and who it belongs to. This matches Notion ("My Templates"), Monday.com ("Your templates"), and Asana ("Templates").

Proposal B: Save Confirmation Dialog

Pattern source: Asana's save dialog lets you choose what to include. Monday.com asks for name + description. We combine both: confirm the name/description (pre-filled from the template) and show a read-only step preview so users know exactly what they're saving.
Key decision: No "share with team" toggle for now. Personal only — keeps the scope minimal and avoids permission complexity. Can be added in a future iteration.

Proposal C: "Add Template" Dialog with Tabs

Pattern source: GitHub Actions' "New workflow" page uses a two-section layout (org templates at top, marketplace below). Monday.com's Template Center uses tabs ("Your templates" vs "From monday.com"). Notion's template gallery has "My templates" vs community.
Design decision: Rename and restructure the import dialog as "Add Template" with three tabs: "My Templates" (personal), "Platform Templates" (admin-curated), and "Create New" (build from scratch). This unifies all template addition into a single entry point.
Smart default tab: If the user has saved templates, the dialog opens on "My Templates". If they have none, it opens on "Platform Templates" — never show an empty tab as the first thing they see.

Proposal D: Single "+" Button Replaces Two Icons

Problem: The current sidebar has two small icons — a book (import) and a plus (create). This is "icon soup": users hover over both trying to figure out which does what, or they always pick the one they tried first and never discover the other. Two cryptic icons in a tight header is a classic discoverability failure.
Solution: Replace both with a single "+" button that opens the unified "Add Template" dialog (Proposal C). The dialog's three tabs handle all paths: import from My Templates, import from Platform Templates, or Create New. This matches Linear (one "Create" button → template picker with blank option) and Notion ("New" → shows templates + blank).
Benefit: One entry point, zero confusion. The "+" is the universal "add something" affordance — users don't need to decode icons.

Technical Decisions

Copy vs Reference

Approach Pros Cons Verdict
Copy on save + Copy on import Simple. No cross-idea deps. Consistent with existing platform template import. Users can edit their per-idea copy freely. Edits to the original don't propagate. Users must manually re-save if they improve a template. ✓ Recommended
Reference (live link) Edits propagate. Single source of truth. Complex: cross-idea RLS, cascade deletes, version conflicts, confusing "who owns this?". Breaks the existing per-idea isolation model. ✕ Too complex

Database Schema Addition

-- New table: personal saved templates
CREATE TABLE user_workflow_templates (
id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
user_id UUID NOT NULL REFERENCES auth.users(id),
name TEXT NOT NULL,
description TEXT,
steps JSONB NOT NULL, -- WorkflowTemplateStep[]
source_idea_id UUID REFERENCES ideas(id) ON DELETE SET NULL, -- where it was saved from
source_idea_title TEXT, -- denormalized for display after idea deletion
suggested_label_name TEXT,
suggested_label_color TEXT,
created_at TIMESTAMPTZ DEFAULT now(),
updated_at TIMESTAMPTZ DEFAULT now()
);

-- RLS: users can only see/manage their own templates
ALTER TABLE user_workflow_templates ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users manage own templates"
ON user_workflow_templates
FOR ALL USING (auth.uid() = user_id);
