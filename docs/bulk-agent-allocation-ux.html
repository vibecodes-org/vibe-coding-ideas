UX Design: Bulk Agent Allocation

Agent: Compass (UX Designer) Task: Make it easy to apply all team members to an idea Date: 2026-03-22

1. Summary

Building on the Requirements step, this design adds two complementary ways to bulk-add agents: an "Add All" button for the common "I want all my agents here" case, and multi-select checkboxes in the popover for selective bulk adds. Both use the same backend action and share a consistent interaction pattern.

Design decision: Requirements R2 mentioned keeping "single-click to add" as a fallback. After consideration, I'm recommending dropping single-click mode in favour of the checkbox approach everywhere. Rationale: maintaining two interaction modes in the same popover is confusing (does clicking the row check or add?). The checkbox + "Add Selected" pattern is clear, consistent, and only adds one extra click for the single-agent case. The "Add All" header button covers the speed case.

5. Interaction Flows

Flow A: "Add All" Button

User sees "Add All (5)" button │ ▼ Clicks button │ ▼ Button shows loading spinner (disabled during operation) │ ▼ allocateAllAgents(ideaId) — bulk insert, single rematch │ ▼ Toast: "Added 5 agents to pool" │ ▼ Agent grid refreshes "Add All" button disappears (0 unallocated remaining)

Flow B: Multi-Select Popover

User clicks "Add Agent" button │ ▼ Popover opens with checkbox list (all unchecked initially) │ ├──► "Select all" link toggles all on/off │ ▼ User checks desired agents │ ▼ Footer shows "Add N Agents" (N = checked count) Button disabled when N = 0 │ ▼ User clicks "Add N Agents" │ ▼ allocateAllAgents(ideaId, selectedBotIds) │ ▼ Toast: "Added N agents to pool" Popover closes Grid refreshes

7. Accessibility

  • Keyboard: Tab through checkboxes, Space to toggle, Enter on "Add Selected"
  • Screen reader: Checkboxes use role="checkbox" with aria-checked; "Select all" is a button with descriptive label
  • Focus trap: Popover uses Radix Popover (existing) which manages focus trap automatically
  • Colour: Checked state uses accent purple fill (4.5:1 contrast on dark bg). Unchecked border is visible at 3:1 minimum
  • Loading: Button text changes to indicate loading state for screen readers

8. Mobile

No special mobile treatment needed. The popover already uses Radix which adapts to viewport. The "Add All" button is a standard button that works at any width. The checkbox targets are 44px touch-friendly (full-row clickable, not just the checkbox).

10. Design Decision: Drop Single-Click Mode

Recommendation: Remove AC9 from requirements

AC9 states: "Existing single-add via popover still works (click agent without checkbox = immediate add)." This creates a confusing dual-mode interaction in the popover. Instead:

  • The popover always uses checkboxes + "Add Selected"
  • For quick single-add, user checks one agent and clicks "Add 1 Agent" (2 clicks vs 1, acceptable)
  • The "Add All" header button covers the speed case for bulk adds
  • This simplifies the implementation and avoids UX confusion
