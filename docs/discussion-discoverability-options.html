Discussion Discoverability — UX Options

4 approaches to surface discussions as an upstream planning activity, between Idea and Board in the workflow.

Design principle: Discussions are an upstream planning activity that feeds INTO the board. They should NOT appear on the board page tabs. The board tabs (Board, Workflows, Agents) are execution tools. Discussions live between the Idea and the Board in the workflow: Idea → Discussions → Board.

Option A: "Recent Discussions" Preview on Idea Detail Page

Replace the simple "Discussions" button with a richer inline section showing recent threads, pinned items, and a quick-start empty state.

Rationale

The idea detail page is the natural "home base" before entering the board. Surfacing discussions here reinforces the Idea → Discussion → Board flow and makes them impossible to miss.

Effort

Medium

Requires fetching recent discussions with a query, new section component, empty state, and adjusting the existing idea detail layout.

Trade-offs

  • Adds vertical height to the idea page
  • Needs a data fetch even when no discussions exist
  • Replaces the simple button — slightly heavier first impression

Option B: "Discussions" Link in Board Page Breadcrumb

Add a secondary breadcrumb link on the board page so users can jump to discussions without going back through the idea page first.

Rationale

Users on the board may need to reference or start discussions. This link is a low-friction escape hatch back to the planning layer, reinforcing the upstream relationship without polluting the board tabs.

Effort

Low

A single link added to the existing breadcrumb. Requires only the discussion count from the idea data already loaded.

Trade-offs

  • Very subtle — easy to overlook
  • Shows count but no preview of content
  • Only helps users who are already on the board

Option C: Idea Card Discussion Preview

When an idea has discussions, show the most recent discussion title as a truncated preview line on the idea card in the feed.

Rationale

The idea feed is the entry point for discovery. Showing an active discussion teaser creates curiosity and signals that the idea has engaged planning activity — social proof that draws users in.

Effort

Low

Requires joining the latest discussion title in the idea feed query and a conditional render line on the card. Minimal UI work.

Trade-offs

  • Only shows the latest — no sense of overall discussion activity
  • Adds visual noise to cards that already have stats
  • Title alone may not convey enough context

A compact widget on the user's dashboard showing discussions with recent activity across all their ideas.

Effort

Medium

New dashboard component, cross-idea discussion query with activity ordering, and direct navigation links to individual threads.

Decision Matrix

Option Impact Effort Recommendation
A. Recent Discussions on Idea Page High — puts discussions front and center on the natural entry point; reinforces Idea → Discussion → Board flow Medium Do now
B. Breadcrumb Link on Board Low-Med — useful escape hatch for board users, but subtle and easy to miss Low Do now
C. Card Discussion Preview Low — nice social proof signal on the feed, but minor and only shows one title Low Defer
D. Dashboard Active Discussions Med-High — drives engagement across all ideas, but competes for dashboard space Medium Defer
Recommended approach: Implement A + B together. Option A is the highest-impact change — it makes discussions visible right where users plan their work. Option B is nearly free and provides a useful shortcut from the board back to the planning layer. Options C and D are nice-to-haves that can be added later once discussion adoption grows.
