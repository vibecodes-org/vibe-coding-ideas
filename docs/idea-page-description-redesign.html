Idea Page Description Redesign

The description has the weakest visual presence on the page despite being the primary content. Three problems to solve: heading hierarchy, page ordering, and long-content handling.

Problem Analysis

Current heading hierarchy (broken):

SectionStyleVisual Weight
Commentstext-lg font-semiboldStrong
Discussionstext-sm font-semibold + icon + cardStrong
Attachmentstext-sm font-semibold + iconStrong
Descriptiontext-[11px] uppercase text-muted-foreground/60Weakest

Current page order: Getting Started → Discussions → Description → Attachments → Comments

The description is sandwiched between Discussions and Attachments, with weaker visual treatment than both.

The dilemma: Moving description above Discussions fixes ordering but long descriptions push everything else off-screen. We need a solution that works for both 1-line and 20-paragraph descriptions.

Proposed Solution

Three changes that work together:

  1. Match the heading style — Description gets the same text-sm font-semibold + icon treatment as Discussions and Attachments
  2. Move above Discussions — Description is primary content, should be first after Getting Started
  3. Collapse long descriptions — Max-height of ~120px with gradient fade + "Read more" link. Short descriptions show fully. This prevents long content from pushing Discussions off-screen.

Implementation Details

Change 1: Description heading (inline-idea-body.tsx)

Replace the current tiny label:

text-[11px] uppercase tracking-wide text-muted-foreground/60

With the standard section heading pattern (matching Attachments):

<h3 className="flex items-center gap-2 text-sm font-semibold">
  <FileText className="h-4 w-4" />
  Description
</h3>

Change 2: Page order (ideas/[id]/page.tsx)

Move the description card (lines 495-503) above the Discussions Preview (lines 485-493). Also separate GitHub URL from Description — extract it above the description or as its own small section.

Change 3: Collapsible long descriptions (inline-idea-body.tsx)

Add collapse/expand behaviour:

  • Default max-height: 120px (~5 lines) with gradient fade overlay
  • "Read more ↓" link below the collapsed content
  • Clicking expands to full height, link changes to "Show less ↑"
  • Only collapse if content height exceeds threshold (use useRef + useEffect to measure)
  • Editing always shows full content (no collapse while editing)
  • State: isExpanded (default false), isOverflowing (measured on mount/resize)

Change 4: GitHub URL separation

Move the GitHub URL rendering outside the description card. It becomes its own small inline element below the description — a simple link pill, not mixed into the description component.
