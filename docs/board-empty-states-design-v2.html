Design Rationale

This revision stays within VibeCodes' existing visual language:

  • Dashed icon circles — matches the agents empty-state pattern (border-2 border-dashed bg-{color}/5)
  • Tinted tile cards — matches welcome-experience action tiles (border-{color}-500/20 bg-{color}-500/5)
  • Solid primary buttons — standard white-on-dark, no gradients
  • Atmospheric blurs at 3-4% — same opacity as welcome card and landing hero
  • Accent colours are informational — emerald = agents, amber = workflows, violet = AI/generation. Consistent across all surfaces.

No gradient fills, no coloured shadows, no glow effects. The colour comes from tinted borders, tinted backgrounds at low opacity, and the accent colour on icon containers. This matches how the rest of VibeCodes uses colour.
