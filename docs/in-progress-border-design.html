Animation Spec

CSS Animation: pulse-blue

  • Duration: 2s, ease-in-out, infinite
  • 0% / 100%: border-color: rgba(59,130,246,0.4), box-shadow: 0 0 8px rgba(59,130,246,0.15)
  • 50%: border-color: rgba(59,130,246,0.7), box-shadow: 0 0 16px rgba(59,130,246,0.3)
  • Subtle pulse — the border brightens and the glow expands slightly, then fades back
  • 2s cycle is slow enough to not be distracting, fast enough to notice

Tailwind Implementation

The animation can't be fully expressed with Tailwind utility classes — needs a custom @keyframes in the component or global CSS. Recommended approach:

  • Add the animate-pulse-blue class with inline style for the keyframes, OR
  • Define the keyframe in Tailwind config / global CSS and reference via class
  • Static classes: border-blue-500/40 ring-1 ring-blue-500/20 as the base, with the animation handling the glow pulse

Accessibility

  • Animation respects prefers-reduced-motion: reduce — disable the pulse, keep the static blue border
  • Blue-500 on dark background meets WCAG contrast requirements for decorative borders
  • The border is decorative (status info is also conveyed by the badge text) — not relying solely on colour/animation
