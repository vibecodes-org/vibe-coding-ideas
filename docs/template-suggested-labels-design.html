Also create labels & auto-rules

When a suggested label exists, importing will create the label and an auto-rule that triggers the workflow when the label is applied to a task.

Design decisions:
• The suggested label badge uses a neutral style (not the label's actual color as the full badge) to avoid visual overload — just a small color dot + name.
• The badge appears inline with the template name, not on a separate row, to keep the card compact.
• "Imported" badge takes visual priority (shown first) when both are present.
• The auto-wire checkbox is positioned between the template list and footer — it's a global option, not per-template.

2b. With Auto-Wire Unchecked

When unchecked, the hint text changes and the label badges remain visible (informational only). Import behavior reverts to the current copy-only approach.

Also create labels & auto-rules

Labels and auto-rules will not be created. You can set these up manually later from the Workflows tab.

3. Success Feedback

The toast message changes to reflect what was actually created, giving users confidence about the auto-wiring.

3a. With Auto-Wire (typical case)

✓ Imported 2 templates, 2 labels, 1 auto-rule
Why different counts? Labels created may differ from templates imported (e.g. a label already existed, or a template had no suggested label). Auto-rules may differ too (if one already existed). Show each count individually so the user understands exactly what happened.

4. Admin Template Editor — Suggested Label Fields

The admin template editor gains an optional "Suggested Label" section between the description and steps. This is a lightweight addition — just a name input and a color picker row.

Design decisions:
• The suggested label section is visually distinct (violet highlight border) to draw attention to the new feature without disrupting the existing form flow.
• Color picker uses the same 10 colors as the board label system (LABEL_COLORS constant) — no new colors introduced.
• Both fields are optional — clearing the name removes the suggested label entirely.
• The "NEW" badge is temporary and can be removed after a few weeks.

6. Accessibility Notes

  • Auto-wire checkbox uses a real <label> element for click-to-toggle and screen reader association
  • Color swatches in admin editor have title attributes and are keyboard-navigable
  • Suggested label badge uses aria-label (e.g. "Suggested label: Feature, violet")
  • Toast messages are announced via the existing sonner ARIA live region
  • Color dots use sufficient contrast against the card background (all ≥ 3:1 on dark surface)

7. Implementation Guidance

  • Color picker reuses LABEL_COLORS from src/lib/constants.ts — no new component needed, just map swatches
  • The WorkflowLibraryTemplate type already includes all Row fields; adding two nullable strings is straightforward
  • The auto-wire checkbox state is local to the dialog — no need to persist it
  • Import dialog already receives existingTemplateNames — extend to also pass existing label names for dedup hints (nice-to-have, not required)
  • The importTemplateWithLabel() server action should be atomic per-template (template + label + rule in one call), but partial failure across multiple templates is acceptable
