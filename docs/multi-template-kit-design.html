Multi-Template Kit Design v2

Compass (UX Designer) — Rework addressing Design Review feedback — Building on Requirements step

Rework context: The previous design was rejected because it (1) removed the workflow step chain from Kit Preview, (2) used inconsistent terminology, and (3) only offered one design option. This revision fixes all three issues.

1. Terminology Audit & Recommendation

An audit of all UI surfaces, MCP tools, guide pages, and code found the following usage:

Concept DB / Code UI (what users see) MCP Tool Name / Desc
Reusable step blueprint workflow_templates "Workflow Template" (header), "Template" (sidebar), "Workflow" (shorthand) *_workflow_template
Label → template auto-link workflow_auto_rules "Workflow Trigger" (everywhere in UI) *_auto_rule (name) / "workflow trigger" (desc)
Active run on a task workflow_runs "Workflow" (Apply Workflow, Reset Workflow) claim_next_step etc.
Admin library blueprint workflow_library_templates "Template Library" (imported via UI)

Recommended Standard Terminology

Use these terms consistently in all new UI:
• "Workflow" — the generic concept / the active run on a task
• "Workflow template" — a reusable step blueprint (shortened to "template" when context is clear, e.g. Workflows tab sidebar)
• "Workflow trigger" — a label-to-template mapping that auto-applies a workflow (this is already the standard across all UI surfaces)
• Never use "auto-rule" in user-facing text — that's the internal DB/code term only

The current kit preview uses these terms correctly. The new design must continue using "workflow trigger" (not "auto-rule") and "workflow template" (not just "template" in isolation).

8. Mobile Considerations

All three options work on mobile with the existing responsive patterns:

  • Option A: Trigger rows stack naturally (they're already vertical)
  • Option B: Tabs may need to scroll horizontally or stack. Template names like "Web Application Feature" are long. Consider truncating or using a dropdown instead of tabs on mobile.
  • Option C: Merged rows stack naturally. Label pills may need to be left-aligned without the min-width constraint.

Option A is the safest for mobile since it requires no new interaction patterns.

Recommendation: Option A is the strongest choice. It's the smallest change from the current UI, keeps the step chain, adds clear trigger mappings, works perfectly on mobile, and is the easiest to implement. Users who want to see secondary template steps can find them in the Workflows tab after the kit is applied.
