AI Board Generation — UX Fix

Design spec for eliminating the 15-20s perceived hang after task insertion completes.

Context: Building on the Requirements step (BA Bob). The root cause is the sequential triggerAutoRulesForTasks loop that runs after the progress bar hits 100%. The user sees a frozen dialog with no indication that work is still happening. This design addresses UX feedback (Phase 1) and previews the async offload approach (Phase 3).

1. Current Flow

The existing dialog state machine. The hang occurs between the inserting and loading-board phases — auto-rules run silently after the progress bar reaches 100%.

Problem: After all tasks are inserted, triggerAutoRulesForTasks() runs synchronously inside insertTasksSequentially() (import.ts:986). The progress bar shows 100%, the dialog phase is still "inserting", and no status text updates. The user sees a frozen UI for 15-20 seconds with no feedback.

2. Proposed Flow

Insert a new "wiring-workflows" phase that gives the user continuous feedback during auto-rule processing.

Key change: The DialogPhase type gains "wiring-workflows". The transition from inserting fires immediately after all tasks are inserted (before triggerAutoRulesForTasks begins). A new callback onAutoRulesProgress provides real-time wiring counts. When auto-rules complete, the phase transitions to loading-board as before.

Design rationale: The indeterminate shimmer bar signals "still working" without implying a knowable percentage. The task list stays visible (dimmed) so the user retains context of what was created. Workflow-matched tasks surface with a small "wired" badge, giving a sense of progress without needing exact counts. The spinner in the footer reinforces that the dialog is active, not frozen.

4. Edge Cases

No auto-rules configured

If the board has no auto-rules, skip the wiring phase entirely. The callback reports 0 tasks to wire, so we go straight to loading-board.

5. Phase 3 Preview: Async Offload

In Phase 3 the dialog closes immediately after task insertion. Workflow wiring happens in the background. The board shows a dismissible banner while wiring is in progress, and workflow indicators appear progressively via Realtime.

Phase 3 design notes:
  • Banner auto-dismisses when all wiring completes (via Realtime subscription on task_workflow_steps)
  • Individual task cards show "Wiring workflow..." with a pulsing dot until their workflow is ready
  • Toast fires on completion: "Workflows ready — 8 tasks wired"
  • If wiring fails, banner turns amber: "Some workflows failed to set up" with a retry link
