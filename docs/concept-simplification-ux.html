3. Agent Creation Soft Validation Hints

Building on the Requirements step: show amber inline hints below empty role and system prompt fields when the user submits the create-agent dialog. Non-blocking — the agent is still created.

Create Agent Dialog — Soft Validation
Name *
My New Agent
Role
e.g., Full Stack Engineer
⚠️ Add a role so this agent can be auto-assigned to workflow steps
System Prompt
Describe how this agent should behave...
⚠️ Add a system prompt so this agent knows how to approach tasks

Interaction Rules

  • When shown: On form submit, if role OR system prompt is empty. Hints appear after submission attempt.
  • When hidden: As soon as user types in the field (onChange), the hint for that field disappears.
  • Blocking: No — form still submits. Agent is created. Hints are advisory only.
  • Styling: Amber border on the empty field + hint box below. Uses border-amber-500/40 + bg-amber-500/[0.08] pattern.
  • Accessibility: Hints use role="alert" so screen readers announce them.
