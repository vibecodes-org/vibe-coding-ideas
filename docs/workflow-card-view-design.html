Workflow Card — Idle Chip Redesign

UX Design step (REWORK) · Target: WorkflowStatusBadge in src/components/board/board-task-card.tsx

Per the Requirements step: board task cards should communicate, at a glance, that a workflow is attached and how far along it is — including the workflow's name — before the run begins. The current idle chip shows only a bare grey ☑ N/M, giving neither a name nor a visual sense of progress.

Corrected scope — IDLE STATE ONLY

The prior submission redesigned all six chip states and was rejected as too broad. This rework changes only the idle branch of WorkflowStatusBadge (the chip shown when a workflow is attached but not yet running).

States left UNCHANGED (do not touch):

  • failed — red chip + AlertTriangle + "{title} failed" + separate fraction
  • stale — amber chip + Clock + "{title} stale · {timeLabel}" + separate fraction
  • complete — emerald chip + CircleCheck + fraction

5 · Data gap (must be surfaced by implementation)

The workflow name is not currently on the card task object. board_tasks exposes workflow_step_completed and workflow_step_total (verified in src/types/database.ts), but there is no workflow_template_name field.

Implementation must surface it by joining workflow_runs.template_id → workflow_templates.name onto the same query that already hydrates the workflow_step_* counts, exposing it as e.g. task.workflow_template_name.

Fallback: when the name is absent/null, render the literal string "Workflow". The chip must never render an empty name.

6 · Implementation spec — idle branch only

Markup (replaces only the case "idle" return)

// derived just for the idle branch:
const name = task.workflow_template_name ?? "Workflow";
const pct =
  workflow_step_total > 0
    ? Math.min(100, (workflow_step_completed / workflow_step_total) * 100)
    : 0; // divide-by-zero guard

case "idle":
  return (
    <Tooltip>
      <TooltipTrigger asChild>
        <span
          className="inline-flex max-w-full flex-col gap-0.5"
          role="img"
          aria-label={`Workflow: ${name}, ${workflow_step_completed} of ${workflow_step_total} steps complete`}
        >
          <span className="inline-flex max-w-full items-center gap-1 rounded-full
                           bg-blue-600 px-2 py-0.5 text-[10px] font-semibold text-white">
            <Play className="h-2.5 w-2.5 shrink-0 fill-current" />
            <span className="truncate min-w-0">{name}</span>
            <span className="shrink-0" aria-hidden>·</span>
            <span className="shrink-0 tabular-nums">{fraction}</span>
          </span>
          <span aria-hidden className="h-[3px] w-full overflow-hidden rounded-full bg-blue-500/20">
            <span className="block h-full rounded-full bg-primary" style={{ width: `${pct}%` }} />
          </span>
        </span>
      </TooltipTrigger>
      <TooltipContent>{name} — not started ({fraction})</TooltipContent>
    </Tooltip>
  );
  • Icon: lucide Play with fill-current, sized h-2.5 w-2.5 shrink-0. Add Play to the lucide import; CheckSquare can be removed if unused elsewhere.
  • Chip: inline-flex max-w-full items-center gap-1 rounded-full bg-blue-600 px-2 py-0.5 text-[10px] font-semibold text-white
  • Container: inline-flex max-w-full flex-col gap-0.5 so the bar sits directly under the chip.
  • Progress bar: ~3px tall (h-[3px]), track bg-blue-500/20, fill bg-primary width = pct%. Lightweight inline div — not the shadcn Progress component (avoids extra nodes across many cards).
  • Truncation: name truncate min-w-0; icon, · separator, and fraction all shrink-0 so the fraction is never clipped.
  • Guards: pct = 0 when workflow_step_total === 0; Math.min(100, …) caps any overflow.

7 · WCAG 2.1 AA confirmation

  • Contrast: white text on bg-blue-600 (#2563eb) ≈ 4.9:1 — passes AA for small text (≥4.5:1). bg-blue-500 (#3b82f6) would be only ~3.7:1 and was deliberately rejected.
  • Not colour alone: meaning is carried by the Play icon, the workflow name, and the N/M fraction — not by hue. The progress bar is supplementary.
  • Non-text contrast: the blue bar fill (#3b82f6) on its faint track is a decorative indicator that duplicates the textual fraction, so it is not relied upon as the sole carrier of state.
  • Screen readers: chip wrapper carries role="img" with an aria-label like "Workflow: Feature Development, 0 of 6 steps complete"; the progress bar is aria-hidden to avoid double announcement; the separator · is aria-hidden.
