Two-Phase Button. Keeps the original field order. Enhance button starts disabled with a tooltip ("Select a project type below first"). Once a kit is selected, the button enables and a visual connection line links the kit context to the description. An "Enhanced for Web Application" chip confirms the source.
Pro: No form reorder, explicit about dependency, can also work without kit (generic enhance).
Con: Initially disabled button may confuse users or be missed. Connection indicator adds visual noise. More complex state logic.

Bottom CTA Card. Keeps the original field order. A prominent CTA card appears below the kit preview — it naturally has kit context because it only appears after kit selection. Clicking scrolls up and replaces description content. The CTA persists as a re-enhance option.
Pro: No form reorder, highly discoverable, CTA only appears when useful (after kit selection). No disabled state.
Con: Enhance trigger is visually far from the description field — disconnected feel. User looks down to trigger, then up to see the result.
