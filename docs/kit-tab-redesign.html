Kit Tab Redesign

Compass (UX Designer) — 27 March 2026

Problem

The workflow template tabs look almost identical to the step pills below them. Both use the same rounded-pill shape, similar sizing, and muted styling. Users don't realise the "tabs" are clickable — they just look like another row of labels. The active tab's violet highlight is too subtle against the violet step-pill borders.

Option A — Underline Tabs

Recommended

Classic underline tab pattern. Universally recognised as navigation. The horizontal rule creates a clear boundary between tabs and content. No overlap with the rounded pill shape used by steps.

Pros

Cons

Slightly less visually distinctive than segmented control. Underline can feel dated in some contexts.

Option B — Segmented Control

iOS/macOS-style segmented control. The recessed track background clearly signals "this is a toggle group". The active segment has an elevated card appearance that's visually distinct from the flat step pills.

Pros

Very clear "toggle" affordance. Recessed track + elevated active state is unambiguous. Modern, polished feel. Groups options tightly.

Cons

Can feel cramped with long template names. Less familiar as "tabs" for web users. Wider horizontal footprint than underline.

Option C — Card Tabs with Step Count Badge

Each tab gets a small icon square and a more button-like appearance. The active state uses a violet tint background + border — still a pill shape, but the icon and larger padding make it clearly a different UI element from the step chain.

Option D — Vertical Sidebar Tabs

Templates listed vertically on the left, step chain on the right. Completely different layout axis from the horizontal step chain, making it impossible to confuse tabs with steps.

Pros

Zero confusion — different axis entirely. Step count badges are clean. Works well when template names are long. Clear visual hierarchy.

Cons

Takes more vertical space. Doesn't work well on narrow mobile screens (would need to collapse to horizontal). More complex layout implementation.

Recommendation: Option A (Underline Tabs)

Why: The underline tab is the most universally understood "this is navigation" pattern on the web. The horizontal rule creates a clear visual boundary between the tab row and the step chain content. It's the simplest to implement (maps directly to shadcn/ui's Tabs component with the default style), takes the least space, and works perfectly on mobile. No one has ever looked at an underline tab and wondered whether it was clickable.

Alternative: If you want something more modern/polished, Option B (segmented control) is the runner-up. It has equally clear affordance and a more premium feel, but takes slightly more horizontal space.
