Concurrent Terminals — Auto-Worktree Isolation UX Design

Compass (UX Designer) · Design step · "Git clash when running more than one terminal" · Builds on the Requirements deliverable (ProdOwner) — Option C locked: auto-worktree per concurrent session. FR references below cite that document.
The open decision this design serves. The isolation decision + git worktree add must run locally — the browser cannot see the filesystem. Two candidate homes: (H) native terminal helper (deterministic, clash-proof, but native and heavily in-flight) vs (A) agent-side bootstrap prompt + heartbeat-lock (ships now, testable, but heuristic). Per the PO recommendation, this design targets (A) as primary; §7 lists exactly where the UX differs under (H) so the Design Review gate can pick with the deltas in view. All touchpoint copy below is home-agnostic unless flagged.

1 · Design principles

  1. Invisible until it matters (FR-1, FR-2). The first session sees one extra line at start and nothing else. Isolation only announces itself when it actually happens.
  2. Always answer "where am I?" (FR-5). Every session states its working dir, branch, and primary/worktree status at start — the single most common confusion with worktrees (mirrors how VS Code and git worktree list surface this).
  3. Never destroy work, always say where it went (FR-6). Dirty/unpushed worktrees are kept, and the keep message doubles as a recovery card: path, branch, and the three commands to resume, publish, or discard.
  4. Degrade loudly, block never (FR-8). Non-git folder or a failed worktree add → one prominent warning, then launch proceeds.
  5. Icon + word, never colour alone. Terminal glyphs are paired with text labels (● Primary, ⧉ Isolated worktree, ⚠ Shared folder) — status survives monochrome terminals and screen readers.

2 · User flows

2a · First / single session — nothing changes (FR-1)

userClicks "Launch Claude Code" on the board → terminal opens in ~/projects/acme-app
bootstrap (A)Checks for a live session lock (.vibecodes/session.lock, heartbeat-stamped) → none found → claims primary, writes lock
terminalPrints the one-line primary banner (§3.1) ← the only visible delta vs today
userWorks exactly as before. No new dirs, no new branches.

2b · Second concurrent session — transparent isolation (FR-2)

userLaunches a second terminal on the same folder while session 1 is live
bootstrap (A)Lock exists + heartbeat fresh → isolate: git worktree add ../acme-app.vibe/wt-2 -b vibe/wt-2
terminalcd into the worktree, print the isolation banner (§3.2) — states path, branch, why, and the way back
user / agentWorks normally — shared .git means full history, zero clash with session 1 ← no user action was required at any point
Naming: worktrees live in a sibling folder <repo>.vibe/wt-<n> (never inside the repo — keeps git status and file watchers clean) on branch vibe/wt-<n>. Predictable names make the recovery copy in §3.4 guessable even from memory. Exact scheme is an eng-review detail; the UX contract is only sibling + predictable + echoed in every banner.

2c · Session end — cleanup decision tree (FR-6)

session endsWas this session in a worktree? No → release lock, done (no output).
check 1Uncommitted changes (git status --porcelain non-empty)? Yes → KEEP + recovery card (§3.4)
check 2Unpushed commits (git log @{u}.. or branch never pushed with commits)? Yes → KEEP + recovery card (§3.4)
cleanNo changes, nothing unpushed → git worktree remove + branch deleted if unused → one-line prune notice (§3.5) ← dirty/unpushed is NEVER auto-deleted; deletion is only ever a user command

2d · Degraded — non-git folder or worktree failure (FR-8)

bootstrap (A)Second session detected, but folder is not a git repo or worktree add fails (bare repo, locked index, disk)
terminalPrints the shared-folder warning (§3.3) — names the risk and the concurrent session count ← warn, don't block
user / agentLaunch proceeds in the original folder. Agent bootstrap additionally instructs the agent to avoid concurrent edits (§5).

3 · Touchpoints — exact copy & placement

3.1 · Session-start banner — primary tree (FR-5)

● Primary checkout · ~/projects/acme-app · branch master · only session on this folder

One line, printed before the agent starts. Glyph ● + the words "Primary checkout" — never colour alone. This is the entire FR-1 footprint.

3.2 · Session-start banner — isolated worktree (FR-2, FR-5)

⧉ Isolated worktree · ~/projects/acme-app.vibe/wt-2 · branch vibe/wt-2 Another session is using ~/projects/acme-app, so this one got its own copy (shared git history, separate files). When done: push this branch or open a PR.

Three lines max. Line 1 answers "where am I"; line 2 answers "why"; line 3 plants the merge-back seed early (FR-7). The ⧉ glyph (two overlapping squares — "a copy") is reused everywhere isolation is shown.

3.3 · Degraded warning — isolation unavailable (FR-8)

⚠ Shared folder — isolation unavailable (not a git repository) 2 sessions are using ~/Downloads/scratch. Concurrent edits can overwrite each other. Continuing anyway. Tip: git init (or run one session at a time) to enable isolation.

The parenthetical swaps per cause: (not a git repository) / (git worktree failed: <first line of stderr>). Same three-line shape either way; launch always proceeds.

3.4 · Session-end — worktree kept (FR-6)

⧉ Worktree kept — it has uncommitted changes (3 files) and 2 unpushed commits. Where: ~/projects/acme-app.vibe/wt-2 (branch vibe/wt-2) Resume: cd ~/projects/acme-app.vibe/wt-2 Publish: git push -u origin vibe/wt-2 then open a PR Discard: git worktree remove ~/projects/acme-app.vibe/wt-2 --force (after you're sure)

A recovery card, not an apology: state → location → the three verbs (resume / publish / discard) as copy-pasteable commands. The reason clause adapts: "uncommitted changes (N files)", "N unpushed commits", or both.

3.5 · Session-end — worktree pruned (clean)

✓ Worktree removed — vibe/wt-2 had no changes to keep. Nothing left behind.

One line. Confirms cleanup happened so leftover-folder anxiety never builds — silence here would make users go check by hand.

3.6 · Optional — terminal-dock tab indicator richer under (H)

acme-app ● acme-app ⧉ wt-2 scratch ⚠ shared

Badge = glyph + short word (⧉ wt-2, ⚠ shared), tooltip/aria-label carries the full sentence. Under (A) the tab label is best-effort (agent self-reports via the relay / OSC title, may lag seconds or go stale). Under (H) the helper owns tab chrome and the badge is authoritative — this is the single biggest UX delta between homes (§7). Tabs stay keyboard-reachable via the dock's existing tab order.

5 · Merge-back guidance — the right copy at the right moments (FR-7)

MomentCopy (verbatim)Why here
Worktree banner, line 3 (§3.2)"When done: push this branch or open a PR."Seed the expectation before any work exists — no surprise at exit.
Agent bootstrap directive (A)"When your work is complete, push branch vibe/wt-N and open a PR. Never merge into or edit the primary checkout directly."The agent is the one most likely to act; make the happy path the instructed path.
Session end, kept card (§3.4)"Publish: git push -u origin vibe/wt-N — then open a PR"The moment of maximum intent: work exists, session over, one copy-paste to safety.
Session end, commits already pushed"✓ Branch vibe/wt-N is pushed. Open a PR when ready; then git worktree remove <path>."Closes the loop — tells the user the worktree is now disposable.

6 · Accessibility (WCAG 2.1 AA)

  • Never colour alone: every status is glyph + word ("⧉ Isolated worktree", "⚠ Shared folder", "● Primary checkout", "✓ Worktree removed"). Fully legible in a monochrome terminal.
  • Contrast: banner accents on #0a0c10: green #4ade80 ≈ 10.5:1, cyan #7dd3fc ≈ 10.9:1, amber #fbbf24 ≈ 10.4:1, dim text #6b7484 ≈ 4.6:1 — all ≥ 4.5:1.
  • Screen readers: banners are plain terminal text — read linearly in order (status → path → branch → action). No box-drawing characters in the copy itself; glyphs precede their text label so the label always disambiguates.
  • Keyboard: dock tabs keep the existing role="tab" order; badges carry aria-label with the full status sentence (mockup in §3.6). No new focus traps; no interaction is added to the terminal stream itself.

7 · (H) vs (A) — where the experience differs

Aspect(A) Agent bootstrap + heartbeat-lock primary target(H) Native terminal helper
Banner timingAppears as the agent's first output — a beat after the shell prompt (seconds). Copy identical.Printed by the helper before the shell starts — instant, guaranteed first line.
Concurrency detectionHeartbeat lock file. Stale-lock caveat: a crashed session's lock past its TTL is treated as free — a session launched inside that window could wrongly claim primary. Mitigation: short TTL + the banner always tells the truth about where it is, so the failure is visible, not silent.Helper holds live session state — deterministic, no stale window, clash-proof.
Dock tab badges (§3.6)Best-effort self-report (relay/OSC title); may lag or stale on crash.Helper owns tab chrome — authoritative badges, kept-worktree reminders can persist on the dock after tab close.
Cleanup on crashNo exit hook ran → worktree simply remains; next session start on that folder appends "1 worktree from a previous session is still around — git worktree list" to its banner.Helper can detect dead sessions and show the kept/prune card retroactively.
Degraded (FR-8)Agent prints §3.3 and also receives the behavioural directive (§4 row 4) — the agent adapts its own conduct.Same §3.3 copy from the helper; agent directive still injected via prompt.
ShipsNow — prompt + small lock convention, testable end-to-end today.After helper lands; all copy above ports 1:1, only the emitter moves.
Design position: every touchpoint is written to be emitter-agnostic — adopting (H) later changes who prints the same words, plus two upgrades (authoritative badges, crash-time cleanup cards). Nothing in (A)'s UX becomes throwaway. That makes "A now, H later" a copy-stable migration, which supports the PO's recommendation.
