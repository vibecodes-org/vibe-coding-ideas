Workflow v2 Role-Based Orchestration

Simplified alternative to PR #62 — same execution power, better UX

Core idea: Templates live on the idea, not on agent profiles. Steps declare roles (BA, UX, Dev, QA, Human). When applied to a task, roles auto-match against the idea's agent pool. No orchestrator/worker distinction. Labels drive workflow association. Claude Code executes autonomously.

Removed

  • × Worker / Orchestrator types
  • × Templates on agent profiles
  • × Orchestrator picker on conversion
  • × set_orchestration_agent MCP tool

Added

  • + Idea-scoped workflow templates
  • + Label → template auto-trigger
  • + Role-based step assignment
  • + Template application preview

Kept (from PR #62)

  • ✓ task_workflow_steps table
  • ✓ claim/complete/fail cycle
  • ✓ Human approval gates
  • ✓ Review loops & step comments
