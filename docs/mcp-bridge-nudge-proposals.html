Layout placement: The new MCP card sits in the left column, below the project/board preview card. This is the same position as the current FirstRunMcpCard — it only renders when hasMcpConnection is false (setup step 4 incomplete). Once the user connects, this card disappears and the progress bar fills to step 4. No layout shifts — the card slot simply empties and the left column gets shorter.
Visual weight: The amber background (amber-500/12%) matches the current card's styling and draws attention without competing with the board preview above. The numbered steps add vertical height (~280px vs the current ~150px), but this is acceptable because the card is the primary CTA at this stage and deserves the space.

Proposal A: Stepped Setup Card (Detail) Recommended

Replace the existing FirstRunMcpCard with a stepped setup card that walks users through prerequisite → connect → verify, all inline. No separate page needed.

Addresses audit gaps: #1 (dead-end card), #2 (no prerequisites), #4 (no "what happens next"), #5 (guide link missing), #6 (no suggested first prompt). Gap #3 (mobile) is handled by a sub-component below.

Why this is recommended: The stepped layout uses progressive disclosure (Hick's Law) — each step is a clear, small action. Users who already have Claude Code installed can skip step 1 at a glance. The suggested prompt at the end gives them a clear "what to do next" that the current card completely lacks.

Mobile Viewport — Context-Switch Nudge

On viewports ≤ 640px, show a device-aware message above the steps. Uses useMediaQuery hook already in the codebase.

Proposal B: Minimal Card + Inline Prereq

Keep the existing card structure but add the three most critical missing elements: a prerequisite hint, the guide link, and a suggested first prompt. Lower effort, lower impact.

Recommendation

Go with Proposal A (Stepped Setup Card). It addresses all 6 audit gaps including the mobile context-switch problem. The implementation is contained to 2 files and ~2 hours of work. The stepped layout follows patterns users know from Linear and Notion onboarding, and the mobile-aware banner prevents frustration for users who signed up on their phone.

The banner enhancements (suggested prompt) should ship with either proposal — they're a 10-minute change that improves every MCP touchpoint.
