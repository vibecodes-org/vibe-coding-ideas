Option B — Toolbar Inside Textarea

The enhance button lives in a footer toolbar attached to the bottom of the textarea — like Slack's message composer or Linear's issue editor. The button is always visible but feels part of the input, not a separate CTA. A character count or hint sits on the left.

Inspired by: Slack message composer, Linear issue editor toolbar, GitHub comment box actions

Option C — Floating Button + Ghost Text Hint

A small floating button anchored to the bottom-right corner of the textarea (like a FAB). When the textarea is empty or has minimal text, ghost placeholder text hints at AI enhancement. Keyboard shortcut (Ctrl+E) for power users. The lightest touch — discoverable but unobtrusive.

✓ Strengths

  • Zero additional vertical space
  • Ghost text teaches the feature contextually
  • Keyboard shortcut for power users
  • Floating button is subtle but always visible
  • Cleanest design — doesn't add visual noise

Compass's Recommendation

Option B (Toolbar) is the strongest all-rounder. It's discoverable without being pushy, feels native to the textarea, and follows established patterns from tools developers already use (Slack, Linear, GitHub). It also leaves room for future toolbar actions (markdown help, attach files).

However, a hybrid approach could work well: show Option A's banner card on first visit only (when user has never enhanced before), then switch to Option B's toolbar for returning users. This gives maximum discovery for new users without annoying repeat visitors.
