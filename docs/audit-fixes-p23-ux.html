P2.3: Remaining Audit Fixes — UX Design

Building on the Requirements step — 4 targeted fixes with before/after specs. All changes are CSS/text modifications to existing components.

Fix 1: Retry Button Always Visible on Failed Steps (F21)

WCAG violation: action buttons must not require hover to be discoverable.

Before — hover required
Failed Implementation ← hover to see Retry

opacity-0 group-hover:opacity-100 — invisible on touch

After — always visible
Failed Implementation

text-muted-foreground hover:text-foreground — subtle but always present

CSS change: Remove opacity-0 group-hover:opacity-100 transition-opacity. Replace with text-muted-foreground/70 hover:text-foreground for subtle always-visible styling. Keep existing size="sm" and variant="ghost".

Fix 2: 50-Task Cap Notice (F16)

When AI generates more than 50 tasks, show an amber notice above the preview table.

After — notice shown when truncated
⚠️ Showing maximum of 50 tasks. Run again to generate more.
[ ImportPreviewTable with 50 tasks ]

Implementation: Add rawTaskCount state. Store lastParsed.tasks.length before slicing. Show notice when rawTaskCount > 50. Notice uses existing amber pattern: border-amber-500/20 bg-amber-500/[0.08] text-amber-400.

Only shown when truncated. If ≤50 tasks generated, no notice appears.

Fix 3: "Apply Workflow" Button Prominence (F14)

Before — dashed ghost

variant="outline" border-dashed size="sm" — easily missed

After — coloured outline

variant="outline" border-primary/50 text-primary — visually distinct

CSS change: Remove border-dashed. Add border-primary/50 text-primary hover:bg-primary/10. The violet colour draws the eye without being as heavy as a solid primary button.
